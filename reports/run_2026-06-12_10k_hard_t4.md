## 3. Kiến trúc đã chạy (pipeline train)
```
Ảnh 384 ─► LHP aug [ON]  (p~N(0.5,1/6): crop quanh bbox người scale≥0.5 / ảnh full; eval luôn dùng full)
        ─► Swin-B [LoRA] ─► f_V ─► Pose branch [ON]: ViTPose 17 kpt → MLP → gated-fuse (train + eval)
Caption ─► BERT[0:6] [FROZEN] ─► f_T
   (f_V,f_T) ─► ITC  (identity soft-target theo sequence_id=video+bucket; temp học từ pretrain, clamp [0.001,0.5])
             ─► Smooth-AP  (λ₂ = 0.2)
   Hard-neg: sampler gom CÙNG VIDEO vào batch (group_fraction 0.5 → cặp hard của team data
   nằm chung batch) + sampling softmax(sim/temp) kiểu X-VLM (mask cùng sequence_id)
             ─► Cross-encoder BERT[6:12] [LoRA] ─► ITM head [TRAIN]  (3N cặp: N pos + 2N hard-neg)
   L = 1.0·ITC + 1.0·ITM + 0.2·Smooth-AP        (MLM đã loại theo plan; weighting = fixed)
```

## 4. Thông số model
| | |
|---|---|

## 9. Nhận xét: ĐÃ DÙNG gì / CHƯA DÙNG gì

### ✅ Đã dùng trong run này
- X-VLM 16M pretrain + **LoRA 1.4%** (image + cross), **text frozen** — đúng plan.
- **ITC** identity soft-target (port chuẩn X-VLM, temp pretrain) + **ITM hard-negative** (sampling theo sim/temp) + **Smooth-AP λ=0.2**.
- **Hard-pair của team data** được tận dụng qua sampler gom cùng video vào batch.
- **ViTPose pose branch** (fuse cả train lẫn eval — nhất quán) + **LHP person-bbox crop**.
- Warmup-cosine, early stopping, fp16, grad-norm diagnostic, **evaluator có distractor**, config nhúng trong checkpoint (tái tạo chính xác).

### ❌ Chưa dùng (theo thiết kế tổng — đây là nguồn cải thiện còn lại)
| Hạng mục | Ghi chú |
|---|---|
| **Cross-encoder ITM re-rank** ở inference | Vũ khí #1 của thiết kế; ITM head **đã được train** trong run này nhưng điểm số trên chưa hề dùng nó — chỉ cosine thuần |
| Sinkhorn/DBSN, Ensemble + Min-Max, Gale-Shapley/SCA | Repo inference chưa xây |
| Tune siêu tham số | Mới chạy **1 cấu hình duy nhất** — chưa sweep LR/λ₂/warmup |
| Ablation đối chứng | Đóng góp thật của pose/LHP/Smooth-AP **chưa đo** (chưa có run OFF để so) |
| Scale data | Mới 10K-hard; subset 50K/100K-hard có sẵn |
| Dynamic loss weighting (uncertainty/DWA) | Có sẵn trong code, run này dùng fixed |
| Đa GPU / all-gather | Chạy 1 T4 → ITC chỉ có 15 negative in-batch/step |
| TTA, paraphrase-aug | Chưa |

### ⚠️ Giới hạn của con số (đọc cho đúng)
1. **0.6615 là điểm VAL-B nội bộ** (gallery 1.6K, 621 query) — **không so sánh được** với 77–89 của các paper (protocol/gallery khác) và **không phải** điểm trên test thật của BTC (gallery 36.773 — khó hơn nhiều, điểm tuyệt đối sẽ thấp hơn). Giá trị của VAL-B là **thước so giữa các cấu hình nội bộ**.
2. 1 seed duy nhất — chênh lệch giữa các run sau cần lớn hơn nhiễu seed mới tính là thật.
3. Đây là **baseline đầu tiên**, không phải kết quả cuối. Các cải thiện kỳ vọng (rerank, scale, tune) **không cộng tuyến tính**.

## 10. Việc tiếp theo (theo ROI)
1. Tải và lưu trữ `best.pth` (Output tab) — checkpoint tham chiếu.
2. **2 ablation đối chứng** (~35 phút/run): (a) pose+LHP OFF; (b) λ₂=0 — trả lời các nhánh phụ có giúp thật không.
3. Sweep nhanh: `lr_lora ∈ {1e-4, 2e-4, 5e-4}`, `warmup_epochs=0.5`.
4. **Xây inference: cross-encoder ITM re-rank** trên top-K (tận dụng ITM head đã train) — kỳ vọng cú nhảy lớn nhất.
5. Scale 50K/100K-hard sau khi chốt cấu hình.
